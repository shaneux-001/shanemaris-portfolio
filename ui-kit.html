02.5 — Two-color system

Duo in action

If a second color is going to earn its keep, it needs a job. Here's the duo running side-by-side across the patterns where it matters most — eyebrows, featured cards, hover/active states, and category framing.

Semantic split — three options

Option A · brand vs UI

Accent 1 = brand identity (logo, eyebrows, hero). Accent 2 = UI action (CTAs, hover, focus).

Cleanest separation. Lets you change one without changing the other. Good if you ever want a tonal shift on a campaign page.

Option B · default vs featured ★

Accent 1 for everything by default. Accent 2 reserved for "this matters more" — featured case study, current role, primary CTA.

Most natural fit for a portfolio. Earns attention without overwhelming.

Option C · work vs labs

Accent 1 colors /work and /about. Accent 2 colors /labs and experimental things.

Personality split. The Konami-code labs section gets to feel different — playful vs professional.

02.75 — Accessibility

WCAG 2.1 AA contrast audit

Computed against the actual portfolio palette. AA requires 4.5:1 for normal text, 3:1 for large text (≥ 24px or ≥ 19px bold) and non-text UI controls. AAA requires 7:1 / 4.5:1.

The hairline failure is the most nuanced. Decorative dividers between sections are exempt from 1.4.11 — they're not communicating state. Control borders on inputs are not exempt: when border is the only thing distinguishing the field from the page, it must hit 3:1. Skill chips that look interactive but aren't are a gray area.

Recommendations, in order

  1. Darken muted from #7A7470 → #6B655F. Single-token change, fixes ~80% of the issues. Same warm-gray family, just one step deeper.
  2. Darken error from #D95F4B → #B4452F. Same hue, deeper saturation. Bonus: it's now in the same family as the terracotta accent.
  3. Split the hairline token in two. Keep #E8E4DE for decorative section dividers (they're exempt from 1.4.11 because they're not control state). Add --color-control-border: #6B655F for input fields and any "looks like a control" element.
  4. Eyebrows on tinted cards: either bump font-size to ≥ 24px (qualifies as large text, drops to 3:1), use a darker accent variant on tint backgrounds, or color those eyebrows with ink/muted instead of accent.
  5. Add a visible focus ring. Currently inputs/buttons have no focus state in source — a 3px accent-tint-15 ring is a low-cost win.
  6. Test custom duo presets. The presets I added (Forest, Saffron, Mono ink, etc.) aren't audited — if you commit to one, run it through this matrix first.

Caveats. WCAG contrast is a floor, not a ceiling — passing 4.5:1 doesn't guarantee readable, and failing slightly doesn't mean unusable. But for a portfolio that bills design-systems work, hitting AA is table stakes. The proposed muted/error darkening is so subtle visually that most viewers won't notice the change while screen-reader and low-vision users will.

Buttons swap text color via color: var(--color-base) rather than hard-coded white — in light mode that's near-white on plum (5.0:1), in dark mode it becomes the dark base on lifted plum (5.0:1). Same token, two correct outcomes.

Implementation notes

  1. Default behavior follows OS via @media (prefers-color-scheme: dark).
  2. Manual override: add class="theme-light" or class="theme-dark" to <html>. The CSS uses :root:not(.theme-light) for the system query so an explicit light class wins.
  3. Since every color goes through a CSS variable, a theme toggle UI just needs to add/remove the class on <html> and persist the preference to localStorage.

07 — Voice & tone

How the site sounds

First-person, plainspoken, declarative. Headlines lean into "I" and the verbs do the work — design, build, scale, lead.

"I design systems that scale."

"I design the systems behind great design."

"From grassroots effort to enterprise-scale design infrastructure."

Patterns: short sentences, active voice, no buzzwords, occasional rhythm via parallel structure ("design / build / scale / lead"). Buttons follow the same rule — "View my resume", "Get in touch", "Read full case study".
